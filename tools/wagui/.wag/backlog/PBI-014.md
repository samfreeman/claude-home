# PBI-014: Explicit Message Groups (User-Defined Tags)

## User Story
As a developer, I want to create custom message groups so that I can organize conversations by topic, not just PBI.

## Problem
Not all important discussions map to a single PBI:
- Cross-cutting concerns (security, performance, architecture)
- Exploratory discussions before PBI creation
- Decisions that affect multiple PBIs

Users need a way to explicitly group related messages.

## Acceptance Criteria
- [ ] User can tag messages with custom labels via UI
- [ ] User can start/stop a named group session via UI button
- [ ] Groups can span multiple PBIs
- [ ] UI shows available groups with message counts
- [ ] Messages can belong to multiple groups
- [ ] User can filter messages by group (like PBI filter)
- [ ] Tests written for new code

## Proposed Solutions

### Option A: Message Tagging
User can add tags to individual messages after the fact.

### Option B: Group Sessions
User starts/stops a named capture session.

### Option C: Hybrid (Recommended)
- Support both tagging and sessions
- Sessions auto-tag messages
- User can manually add/remove tags

## Technical Notes

### Data Model
```typescript
interface MessageGroup {
    id: string
    name: string
    tags: string[]           // e.g., ['security', 'auth']
    messageIds: string[]
    createdAt: number
    isActive: boolean        // For session-based capture
}

// Messages get a groups array
interface WagMessage {
    // ... existing fields
    groups?: string[]        // Group IDs this message belongs to
}
```

### MCP Tools (if using sessions)
```typescript
wag_group_start(name: string): GroupId
wag_group_stop(): void
wag_group_tag(messageId: string, tags: string[]): void
```

### Interaction Flows

**Start Group Session:**
1. User clicks "Start Group" button in toolbar
2. Modal appears for group name and optional tags
3. User enters name, adds tags
4. Clicks "Start Capturing"
5. Recording indicator appears
6. All new messages are added to this group
7. User clicks "Stop" to end session

**Tag Individual Message:**
1. User clicks [+] on a message
2. Tag input appears
3. User types tag name or selects existing
4. Tag chip appears on message

**Filter by Group:**
1. User clicks Groups dropdown
2. Selects a group
3. Message list filters to that group
4. Clear filter button appears
