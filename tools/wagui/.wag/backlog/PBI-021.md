# PBI-021: Subtle Tool Use Messages

## User Story
As a developer, I want tool use messages to be visually subtle so that context state changes stand out more prominently in the wagui feed.

## Problem
Currently, tool use messages (from the PreToolUse hook) appear the same as context state changes. This makes the feed visually noisy and harder to scan for important state transitions.

## Acceptance Criteria
- [ ] Tool use messages are smaller and left-aligned (no bubble)
- [ ] Tool use messages are visually distinct from context dividers
- [ ] Context state changes remain prominent (current divider style)
- [ ] Tool use messages should be subtle but still readable
- [ ] Tests written for new code

## Technical Notes

### Identifying Tool Use Messages
Tool use messages come from the hook and have:
- `role: "dev"`
- `type: "context"`
- `content` starts with "Reading:", "Writing:", "Running:", "Searching:", "Grepping:", "Spawning", or "Using:"

### Design Options
1. **Inline text** - Just small gray text, no bubble or divider
2. **Indented list** - Small monospace text, indented under previous message
3. **Collapsed group** - Group consecutive tool uses, show count, expandable

### Suggested Implementation
Add a helper to detect tool use messages:
```typescript
function isToolUseMessage(message: WagMessage): boolean {
    if (message.role != 'dev' || message.type != 'context') return false
    const prefixes = ['Reading:', 'Writing:', 'Running:', 'Searching:', 'Grepping:', 'Spawning', 'Using:']
    return prefixes.some(p => message.content.startsWith(p))
}
```

Then in Message.tsx, render tool use messages differently:
```typescript
if (isToolUseMessage(message))
    return <ToolUseMessage content={message.content} />
```
